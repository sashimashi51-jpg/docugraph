## Въведение
Този проект служи като функционален RAG потребителски интерфейс както за крайни потребители, които искат да задават въпроси (QA) върху своите документи, така и за разработчици, които искат да изградят свой собствен RAG пайплайн.

```text
+----------------------------------------------------------------------------+
| Крайни потребители: Тези, които използват приложенията, изградени с        |
| Docugraph (Използвате приложението като краен продукт)                     |
|     +----------------------------------------------------------------+     |
|     | Разработчици: Тези, които надграждат Docugraph.                |     |
|     | (Имате `import docugraph` някъде във вашия проект)             |     |
|     |     +----------------------------------------------------+     |     |
|     |     | Сътрудници: Тези, които правят Docugraph по-добър. |     |     |
|     |     | (Вие правите PR към това хранилище)                |     |     |
|     |     +----------------------------------------------------+     |     |
|     +----------------------------------------------------------------+     |
+----------------------------------------------------------------------------+
```

### За крайни потребители
- **Изчистен и минималистичен UI**: Лесен за използване интерфейс за RAG-базирани въпроси и отговори.
- **Поддръжка на различни LLM**: Съвместим с API доставчици (OpenAI, AzureOpenAI, Cohere и др.) и локални модели (чрез Ollama и llama-cpp-python).
- **Лесна инсталация**: Прости скриптове, с които да започнете бързо.

### За разработчици
- **Фреймуърк за RAG пайплайни**: Инструменти за изграждане на ваш собствен RAG-базиран пайплайн за документи.
- **Интерфейс с възможност за персонализиране**: Вижте вашия RAG пайплайн в действие с предоставения UI, изграден с Gradio.

## Основни характеристики

- **Хоствайте собствен RAG web-UI**: Поддръжка на вход за много потребители, организиране на вашите файлове в лични/публични колекции, сътрудничество и споделяне на вашите любими чатове с други.
- **Организирайте вашите LLM и Embedding модели**: Поддръжка както на локални езикови модели, така и на популярни API доставчици (OpenAI, Azure, Ollama, Groq).
- **Хибриден RAG пайплайн**: Разумен RAG пайплайн по подразбиране с хибриден (пълен текст и вектори) ретривър (retriever) и пренареждане (re-ranking) за осигуряване на най-добро качество на търсенето.
- **Мултимодална поддръжка за въпроси и отговори**: Извършване на въпроси и отговори (QA) върху множество документи с поддръжка на фигури и таблици. Поддръжка на мултимодално парсване на документи (опции, избираеми от потребителския интерфейс).
- **Разширени цитати с преглед на документа**: По подразбиране системата предоставя подробни цитати, за да се гарантира коректността на отговорите на LLM. Преглеждайте своите цитати (включително техния резултат за уместност - relevant score) директно във вградения PDF четец в браузъра с подчертавания. Предупреждение, когато пайплайнът за извличане върне статии с ниска уместност.
- **Поддръжка на сложни методи за разсъждение (Reasoning)**: Използвайте декомпозиция на въпроса, за да отговорите на вашите сложни/многостепенни въпроси. Поддръжка на базирано на агенти разсъждение с ReAct, ReWOO и други агенти.
- **UI с конфигурационни настройки**: Можете да коригирате най-важните аспекти от процеса на извличане и генериране директно в потребителския интерфейс (включително промптовете).
- **Разширяемост**: Тъй като е изграден на Gradio, вие сте свободни да персонализирате или добавяте каквито и да е UI елементи, както желаете. Също така, целим да поддържаме множество стратегии за индексиране и извличане на документи (като GraphRAG).

## Инсталация

Този проект е оптимизиран специално за **Windows**. За да го стартирате, са ви необходими само няколко клика. Няма нужда от Docker или сложни виртуални машини.

### Системни изисквания
- Windows 10/11
- Python (инсталира се автоматично от нашия скрипт)

### Бърза автоматична инсталация (Препоръчително)

Това е най-лесният начин да стартирате Docugraph. Скриптът автоматично ще инсталира Miniconda, Python 3.10 и всички необходими библиотеки.

1. Изтеглете или клонирайте това хранилище:
   ```cmd
   git clone https://github.com/sashimashi51-jpg/docugraph.git
   cd docugraph
   ```
2. Отидете в папката `scripts` и кликнете два пъти върху **`run_windows.bat`**.
3. Изчакайте инсталацията да приключи. Приложението ще се стартира автоматично и ще отвори браузъра на `http://localhost:7860/`.
4. Входни данни по подразбиране: Потребител: `admin`, Парола: `admin`.

### Ръчна инсталация чрез Conda

Ако предпочитате да управлявате средите си ръчно, можете да използвате традиционния метод:

1. Създайте и активирайте нова среда:
   ```cmd
   conda create -n docugraph python=3.10
   conda activate docugraph
   ```
2. Клонирайте хранилището и инсталирайте библиотеките:
   ```cmd
   git clone https://github.com/sashimashi51-jpg/docugraph.git
   cd docugraph
   pip install -e "libs/docugraph[all]"
   pip install -e "libs/ktem"
   ```
3. Създайте `.env` файл в основната директория, като копирате `.env.example`.
4. Стартирайте уеб сървъра:
   ```cmd
   python app.py
   ```

## Настройка на локален AI модел (Ollama)

За напълно локален/поверителен RAG пайплайн се препоръчва използването на Ollama.

1. Изтеглете и инсталирайте [Ollama](https://ollama.com/download) за Windows.
2. Отворете нов терминал и изтеглете нужните модели:
   ```cmd
   ollama pull llama3
   ollama pull nomic-embed-text
   ```
3. Стартирайте Docugraph и отидете в таба **Resources** -> **Models**.
4. Добавете нов модел (тип OpenAI) със следните параметри:
   - `api_key`: **ollama**
   - `base_url`: **http://localhost:11434/v1/**
   - `model`: **llama3** (за езиков модел) или **nomic-embed-text** (за embedding модел).
5. Влезте в **Settings -> Retrieval Settings** и задайте тези локални модели по подразбиране.

Налични са следните опции:

За да използвате Docling, първо инсталирайте необходимите зависимости в конзолата: 
`pip install docling`

След това изберете съответните товарачи (loaders) в: 
`Settings -> Retrieval Settings -> File loader`

## Персонализиране на вашето приложение

По подразбиране всички данни на приложението се съхраняват в папката `./ktem_app_data`. Можете да архивирате или копирате тази папка, за да прехвърлите вашата инсталация на нова машина.

За напреднали потребители или специфични случаи на употреба, можете да персонализирате тези файлове:
- `flowsettings.py`: Съдържа конфигурацията на вашето приложение.
- `.env`: Предоставя алтернативен начин за конфигуриране на вашите модели и идентификационни данни.

## Добавяне на ваш собствен RAG пайплайн

### Персонализиран пайплайн за разсъждение (Reasoning Pipeline)
Можете да направите бърза корекция на начина, по който работи QA пайплайнът по подразбиране.
Добавете нова `.py` имплементация в `libs/ktem/ktem/reasoning/` и след това я включете във `flowsettings.py`, за да я активирате в потребителския интерфейс.

### Персонализиран пайплайн за индексиране (Indexing Pipeline)
Разгледайте примерната имплементация в `libs/ktem/ktem/index/file/graph`.
